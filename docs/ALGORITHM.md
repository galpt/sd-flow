# Algorithm: From scx_flow CPU Scheduler to Diffusion Scheduler

This document explains the cross-domain adaptation of the **scx_flow** CPU scheduling algorithm into the **sd-flow** noise scheduler for Stable Diffusion.

---

## 1. The Original: scx_flow (CPU Scheduling)

[scx_flow](https://github.com/sched-ext/scx/tree/main/scheds/experimental/scx_flow) is a Linux kernel BPF-based CPU scheduler from the sched-ext project. It is intentionally simple and deterministic:

- **No vruntime/vtime** — unlike virtually every other scheduler, scx_flow doesn't use weighted fair queuing
- **No heuristics** — doesn't reinterpret `nice` values, doesn't detect "latency-critical" tasks
- **No adaptive tuning** — no auto-tuning, no load tracking

Instead, it uses a **budget-driven** approach:

### Budget Lifecycle

1. **Sleep accumulation**: While a task is sleeping, it accumulates budget at roughly 1% of sleep time
2. **Budget refill**: On wakeup, `budget += sleep_ns / 100 * weight / 100`
3. **Tier classification**: Tasks are classified into 4 FIFO tiers based on their budget level:
   - `PRIORITY` (budget ≥ 1500μs) — long-sleeping interactive tasks
   - `NORMAL` (budget ≥ 1000μs) — typical interactive tasks
   - `LOW` (budget ≥ 500μs) — short-sleep or partially exhausted
   - `DEFICIT` (budget < 500μs) — bulk workers, budget exhausted
4. **Rotating dispatch**: A 4-phase rotation cycles which tier is serviced first each round, guaranteeing no tier waits more than 3 dispatch cycles:
   - Phase 0: PRIORITY → NORMAL → LOW → DEFICIT
   - Phase 1: NORMAL → LOW → DEFICIT → PRIORITY
   - Phase 2: LOW → DEFICIT → PRIORITY → NORMAL
   - Phase 3: DEFICIT → PRIORITY → NORMAL → LOW
5. **Slice allocation**: Running tasks get a time slice proportional to their budget (50–350μs)

### Why This Matters

The rotating dispatch provides a **formal starvation guarantee** — no tier can be permanently locked out. Combined with deterministic budget accounting, every scheduling decision is fully auditable.

## 2. The Target: Diffusion Noise Scheduling (EDM / Karras)

The [EDM paper](https://github.com/NVlabs/edm) (Karras et al., 2022) defines the standard noise scheduler used in Stable Diffusion UIs:

### Sigma Schedule

The timesteps (noise levels) are generated by:

```
t_i = (σ_max^(1/ρ) + i/(N-1) · (σ_min^(1/ρ) − σ_max^(1/ρ)))^ρ
```

Where:
- `σ_max` = 80.0 (initial noise level)
- `σ_min` = 0.002 (final noise level before clean image)
- `ρ` = 7.0 (controls step spacing — higher = more steps near low noise)
- `N` = number of steps (default 18)

When `ρ = 7`, the schedule heavily concentrates steps in the low-noise region, where the ODE dynamics change fastest.

### ODE Solver

The sampler uses Heun's 2nd order method:

```
1. Euler prediction:  x_next = x + (σ_next − σ_cur) · (x − D(x, σ_cur)) / σ_cur
2. Heun correction:   x_next = x + (σ_next − σ_cur) · ½(d_cur + d_prime)
```

Where `D(x, σ)` is the denoiser network output.

## 3. The Adaptation: sd-flow

### Core Insight

Both problems share a fundamental structure: **allocating limited compute across multiple competing entities**.

| CPU Scheduling | Diffusion Sampling |
|---|---|
| Tasks compete for CPU time | Timesteps compete for ODE evaluations |
| Budget accumulates during sleep | Budget accumulates across sigma transitions |
| Tier classification by budget level | Tier classification by accumulated budget |
| Budget determines time-slice length | Budget determines solver order (Heun vs Euler) |

### The Flow Budget for Diffusion

For each sigma transition `σ_i → σ_{i+1}`:

```
Δσ = σ_i − σ_{i+1}
refill = (Δσ / σ_max) × (σ_i / σ_max) × 4.0
budget[i+1] = clamp(budget[i] + refill, B_min, B_max)
```

Where:
- `B_max = 2.0` (maximum budget ceiling)
- `B_min = −0.5` (minimum budget floor, allows slight deficit)
- The `× 4.0` scale factor ensures cumulative budget naturally spans `[0, BUDGET_MAX]`, giving meaningful separation between all 4 tiers.

Large sigma drops at high noise levels generate the most budget — early denoising steps have the most "creative freedom."

### Tier Classification

Each timestep is classified by its budget:

| Tier | Budget | Effect on ODE Solver |
|---|---|---|---|
| PRIORITY | ≥ 1.5 | DDIM step (1 NFE, deterministic) |
| NORMAL | ≥ 1.0 | DDIM step (1 NFE, deterministic) |
| LOW | ≥ 0.5 | Euler Ancestral (1 NFE + noise) |
| DEFICIT | < 0.5 | Euler Ancestral (1 NFE + noise) |

### Per-Step Tier Labels (Not Rotating Dispatch)

The rotating dispatch concept from scx_flow determines which TIER of task runs next. In diffusion, sigma values must be monotonically decreasing (time moves forward), so we cannot reorder steps. Instead, the flow algorithm assigns a **tier label** to each step via budget accumulation:

1. Each sigma transition `σ_i → σ_{i+1}` generates a budget refill proportional to the sigma drop
2. The cumulative budget determines the step's tier (PRIORITY/NORMAL/LOW/DEFICIT)
3. Higher-budget steps use DDIM (deterministic); lower-budget steps use Euler Ancestral (adds noise)
4. If any tier has zero steps (no step accumulated enough budget), the highest-budget step is demoted to that tier to ensure no sigma range is entirely starved of correction

### Schedule Generation

```
Input: num_steps, sigma_min, sigma_max

1. Generate linear sigma spacing: linspace(sigma_max, sigma_min, num_steps)
2. For each transition, compute budget refill:
     refill = (Δσ / σ_max) × (σ_cur / σ_max) × 4.0
     budget[i+1] = clamp(budget[i] + refill, B_min, B_max)
3. Classify each step into a tier by its budget
4. Ensure every viable tier has at least 1 correction step
5. Append trailing 0

Output: torch.Tensor [sigma_max, ..., sigma_min, 0]
         self.step_tiers: list[int]  (per-step tier indices)
```

## 4. Key Differences from Standard Schedulers

| Property | Karras / Normal | Flow Schedule |
|---|---|---|
| Sigma spacing | Karras (polynomial) or Normal (linear) | Linear (same as Normal) |
| Step behavior | Same solver for every step | Adaptive: PRIORITY steps get DDIM, DEFICIT steps get Euler Ancestral |
| Control mechanism | Single parameter `ρ` (Karras) or none (Normal) | Budget thresholds + tier classification |
| Fairness guarantee | None | Every tier gets at least 1 correction step |
| Determinism | Yes | Yes |

## 5. Parameter Reference

| Parameter | Default | Range | Effect |
|---|---|---|---|
| `num_steps` | 18 | [1, 10000] | Total number of sampling steps |
| `sigma_min` | 0.002 | (0, sigma_max) | Lowest noise level |
| `sigma_max` | 80.0 | (sigma_min, ∞) | Highest noise level |
| `budget_max` | 2.0 | [0.1, 10.0] | Maximum budget ceiling |
| `budget_min` | −0.5 | [−2.0, 2.0] | Minimum budget floor |
| `tier_thresholds` | (1.5, 1.0, 0.5) | varies | Budget boundaries for PRIORITY/NORMAL/LOW |

### Tier Threshold Presets

| Preset | Priority | Normal | Low | Effect |
|---|---|---|---|---|
| Default | 1.5 | 1.0 | 0.5 | Balanced DDIM vs Euler Ancestral split |
| Aggressive | 2.0 | 1.5 | 1.0 | More Euler Ancestral steps (higher bar for DDIM) |
| Gentle | 1.2 | 0.8 | 0.4 | More DDIM steps (lower bar for DDIM) |
